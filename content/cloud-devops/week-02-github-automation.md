# Week 2 — GitHub Advanced & Automation

## Objective

Understand **collaboration governance and automation**.

Students will learn how real engineering teams manage large repositories.

---

# Topics

## GitHub Project Management

GitHub includes tools for managing engineering tasks.

Features:

• Issues  
• Labels  
• Milestones  
• Project boards  

Example issue:

Title: Add login endpoint

Description:
Create API endpoint for authentication.

Acceptance Criteria:

* JWT authentication
* password hashing

---

## Pull Request Workflow

Standard engineering workflow:

```

feature branch
↓
pull request
↓
code review
↓
merge

```

Students practice:

• code reviews  
• commenting  
• approval process  

---

## Branch Protection

Production branches must be protected.

Rules include:

• require pull request  
• require review  
• require CI passing  

---

## GitHub Pages

Students deploy static websites.

Example use:

• portfolio site  
• documentation  
• project demo  

Deployment flow:

```

GitHub repo → GitHub Pages → public site

```

---

## Webhooks

Webhooks notify external systems.

Example:

```

GitHub → CI server → deployment

```

Students learn event triggers.
