# Week 10 — Monitoring & Observability

## Objective

Learn how to monitor production systems.

Students will learn:

• logging  
• metrics  
• alerting  

---

# Observability Components

Production systems require:

• logs  
• metrics  
• traces  

---

# CloudWatch Logs

CloudWatch stores application logs.

Example logs:

```

API request logs
error logs
container logs

```

---

# Metrics

Metrics track system performance.

Examples:

• CPU usage  
• request latency  
• error rate  

---

# Alerts

Alerts notify engineers when problems occur.

Example:

```

CPU > 80%

```

Alert triggers notification.

---

# Dashboards

Dashboards visualize metrics.

Example metrics:

• traffic  
• error rate  
• latency  

---

# Cost Monitoring

Track cloud costs using:

• AWS budgets  
• cost explorer
